# НямКа — музыкальный бот с двумя каналами

Как это работает:
1. Кидаешь ссылку в канал **"только ссылки"** — всё, что не ссылка, бот сразу удаляет.
2. Ссылка становится в очередь.
3. Когда доходит её очередь — бот удаляет сообщение из первого канала и публикует его во втором канале со статусом `▶️ Сейчас играет`.
4. Когда трек закончился, это же сообщение меняется на `✅ Отыграно`.

## Установка

1. Установи [Node.js](https://nodejs.org/) (версия 18 или новее).
2. В папке проекта выполни:
   ```bash
   npm install
   ```
3. Скопируй `.env.example` в `.env`:
   ```bash
   cp .env.example .env
   ```
4. Заполни `.env`:
   - **DISCORD_TOKEN** — вкладка "Бот" в Discord Developer Portal → Reset Token → скопировать.
   - **VOICE_CHANNEL_ID**, **LINKS_CHANNEL_ID**, **PLAYED_CHANNEL_ID** — включи режим разработчика в Discord (Настройки → Дополнительно → Режим разработчика), затем ПКМ по нужному каналу → "Копировать ID".

5. В Developer Portal, на вкладке **"Бот"**, включи **MESSAGE CONTENT INTENT** — без него бот не сможет читать текст ссылок.

6. Запусти бота:
   ```bash
   npm start
   ```

## Права при генерации ссылки-приглашения (OAuth2)

- Scope: `bot`
- Права бота: Отправлять сообщения, Читать историю сообщений, Управлять сообщениями (нужно для удаления не-ссылок и старых сообщений), Подключаться, Говорить

## Важный нюанс

YouTube регулярно усложняет жизнь сторонним библиотекам вроде `play-dl` (капчи, блокировки по IP и т.д.), поэтому воспроизведение с YouTube иногда может обрываться или переставать работать без предупреждения — это известная проблема всех самописных ботов, а не баг в этом коде. Если начнутся проблемы именно с YouTube — можно попробовать SoundCloud-ссылки (`play-dl` их тоже поддерживает) или обновить `play-dl` до последней версии.

## Возможные доработки
- Сделать очередь персистентной (сохранять в файл/базу), чтобы не терялась при перезапуске бота.
- Добавить команду `/skip`, `/queue` для контроля прямо из чата.
- Добавить проверку, что бот уже не играет этот же трек (защита от дублей).
